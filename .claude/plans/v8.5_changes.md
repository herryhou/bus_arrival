# v8.5 技術報告變更計劃

## 概述

根據 `docs/bus_arrival_tech_report_v8.md` v8.5 版本，共 15 項修正需要實作：

**功能行為與 v8.4 相同，二進制格式（VERSION: 2）不變。**

## 🔴 嚴重修正（Runtime Bug）

### 1. `RouteNode` repr 修正（Section 5.5）

**檔案：** `shared/src/lib.rs`

**問題：** `repr(C, packed)` 在已有手動 `_pad` 欄位的情況下，對 field 取 `&` 是 UB（未定義行為）。

**變更：**
```rust
// 第 67 行，從：
#[repr(C, packed)]
pub struct RouteNode {

// 改為：
#[repr(C)]
pub struct RouteNode {
```

**理由：** 手動的 `_pad` 欄位已經確保了正確的對齊，`packed` 會移除所有對齊導致 UB。

### 4. 廊道重疊保護基準修正（Section 12.4）

**檔案：** `preprocessor/src/stops.rs`

**問題：** 原公式導致相鄰廊道仍有 20 m 重疊。

**變更：** 第 32-36 行，修正截斷點基準：

```rust
// 從：
let min_separation = 2000; // 20m
let min_start = prev.corridor_end_cm + min_separation;

// 改為：
let min_start = prev.corridor_end_cm + 2000; // 直接使用 2000 cm (20m)
```

**理由：** 文件說明應該是 `corridor_end[i] + δ_sep`，其中 `δ_sep = 2000 cm`。

**檔案：** `shared/src/lib.rs` 和 `arrival_detector/src/state_machine.rs`

**問題：** `Departed` 狀態時 `current_stop_index++` 在末站會 u8 溢位回到 0。

**變更：**

#### 5.1 新增 `TripComplete` 狀態

在 `shared/src/lib.rs` 的 `FsmState` enum 中新增：

```rust

pub enum FsmState {
    Approaching,
    Arriving,
    AtStop,
    Departed,
    TripComplete,  // 新增：行程完成
}
```

#### 5.2 更新 state machine 邏輯

在 `arrival_detector/src/state_machine.rs` 的 `update()` 方法中新增邊界檢查：

```rust
FsmState::Departed => {

    }

FsmState::TripComplete => {
    // Terminal state - no further transitions
```

### 11. Module ⑥ 拒絕後行為明確定義（Section 9.2）

**類型：** 註釋更新

**變更：** 在 `simulator/src/kalman.rs` 中新增註釋說明拒絕後的行為：

```rust
/// Reject GPS updates that exceed physical limits
/// When rejected, only Kalman predict step is executed
/// (equivalent to short-term dead-reckoning, v_cms not updated)
fn check_speed_constraint(z_new: DistCm, z_prev: DistCm, dt: i32) -> bool {
    let dist_abs = (z_new - z_prev).unsigned_abs() as i32;
    let max_dist = V_MAX_CMS * dt.max(1) + SIGMA_GPS_CM;
    dist_abs <= max_dist
}
```

### 12. Kalman `v_cms` 限制非負（Section 10.4）

**檔案：** `shared/src/lib.rs`

**問題：** GPS 雜訊可能產生負速度，導致 predict step 倒推 `ŝ`，觸發連鎖拒絕。

**變更：** 在 `update()` 和 `update_adaptive()` 方法末尾加上 `.max(0)`：

```rust
// 第 158-163 行，從：
pub fn update(&mut self, z_cm: DistCm, v_gps_cms: SpeedCms) {
    let s_pred = self.s_cm + self.v_cms;
    let v_pred = self.v_cms;
    self.s_cm = s_pred + (51 * (z_cm - s_pred)) / 256;
    self.v_cms = v_pred + (77 * (v_gps_cms - v_pred)) / 256;
}

// 改為：
pub fn update(&mut self, z_cm: DistCm, v_gps_cms: SpeedCms) {
    let s_pred = self.s_cm + self.v_cms;
    let v_pred = self.v_cms;
    self.s_cm = s_pred + (51 * (z_cm - s_pred)) / 256;
    self.v_cms = (v_pred + (77 * (v_gps_cms - v_pred)) / 256).max(0);
}

// 第 165-171 行，從：
pub fn update_adaptive(&mut self, z_cm: DistCm, v_gps_cms: SpeedCms, hdop_x10: u16) {
    let ks = Self::ks_from_hdop(hdop_x10);
    let s_pred = self.s_cm + self.v_cms;
    let v_pred = self.v_cms;
    self.s_cm = s_pred + (ks * (z_cm - s_pred)) / 256;
    self.v_cms = v_pred + (77 * (v_gps_cms - v_pred)) / 256;
}

// 改為：
pub fn update_adaptive(&mut self, z_cm: DistCm, v_gps_cms: SpeedCms, hdop_x10: u16) {
    let ks = Self::ks_from_hdop(hdop_x10);
    let s_pred = self.s_cm + self.v_cms;
    let v_pred = self.v_cms;
    self.s_cm = s_pred + (ks * (z_cm - s_pred)) / 256;
    self.v_cms = (v_pred + (77 * (v_gps_cms - v_pred)) / 256).max(0);
}
```
