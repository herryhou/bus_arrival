# Bus Arrival Simulator (Localization Pipeline)

The `simulator` crate implements the **Phase 2: Localization Pipeline** of the bus arrival detection system. It processes raw GPS data (in NMEA format) and maps it onto a pre-processed route to estimate the bus's progress along that route in centimeters.

## Overview

The simulator takes NMEA logs and binary route data (generated by the `preprocessor`) to produce a precise estimate of the bus's position (`s_cm`) and speed (`v_cms`). It employs heading-constrained map matching and an adaptive Kalman filter to handle GPS noise and outages.

### Key Features

- **NMEA Parsing**: Supports `$GPRMC`, `$GPGGA`, and `$GNGSA` sentences for position, speed, and precision (HDOP) data.
- **Heading-Constrained Map Matching**: Uses both distance and bus heading to select the most likely route segment, preventing "jumping" to nearby parallel streets or overlapping loops.
- **Adaptive Kalman Filter**: Smoothes progress estimates based on GPS quality (HDOP) and physical constraints.
- **Dead Reckoning (DR)**: Automatically switches to dead reckoning during short GPS outages (up to 10 seconds) using the last known filtered speed.
- **Physical Constraints**: Filters out impossible GPS jumps using speed and monotonicity constraints.

## Usage

The simulator is a CLI tool that processes a single NMEA log against a specific route.

```bash
cargo run --bin simulator <nmea_file> <route_data.bin> <output.jsonl>
```

### Arguments

1. **`nmea_file`**: Path to a file containing raw NMEA sentences.
2. **`route_data.bin`**: The binary route file produced by the preprocessor (contains nodes, stops, and spatial grid).
3. **`output.jsonl`**: The output file where localization results will be written in JSON Lines format.

## Technical Architecture

### 1. NMEA Processing (`nmea.rs`)
The parser identifies and extracts coordinates, speed, and heading from `$GPRMC`/`$GPGGA`. It uses `$GNGSA` or `$GPGGA` to extract HDOP, which informs the Kalman filter's measurement noise. Heading values are converted to a standardized centi-degree range (-18000 to 18000) to avoid overflow.

### 2. Map Matching (`map_match.rs`)
To efficiently find the bus's position on the route:
- **Spatial Grid**: Uses the pre-calculated grid in the binary file for fast lookups.
- **Heading Weighting**: When the bus is moving, it prefers segments that align with the GPS heading.
- **Windowed Search**: Prioritizes segments near the last known position to maintain continuity.

### 3. Localization Pipeline (`kalman.rs`)
For every GPS update, the pipeline follows these steps:
1. **Coordinate Projection**: Converts Lat/Lon to absolute cm coordinates relative to a fixed origin (120°E, 20°N).
2. **Map Match & Project**: Finds the best segment and projects the point onto it to get raw progress (`z_raw`).
3. **Filter**: Validates the update against a maximum speed (`V_MAX_CMS`) and allows for minor backward jitter (up to 50m) for noise tolerance.
4. **Kalman Update**: Updates the state using an HDOP-adaptive gain.
5. **DR Fallback**: If a fix is lost, it projects progress forward based on the last valid speed for up to 10 seconds.

### 4. Output Generation (`output.rs`)
The resulting `output.jsonl` provides a detailed trace of the localization process, including:
- `s_cm`: Cumulative progress along the route in centimeters.
- `v_cms`: Filtered speed in cm/s.
- `status`: Result of the update (e.g., `"valid"`, `"rejected_speed"`, `"dr_outage"`).
- `active_stops`: Indices of stops whose "corridors" the bus is currently within.
- `stop_states`: Distance to each active stop.

## Testing

The crate includes extensive tests covering:
- **Coordinate Alignment**: Ensures the simulator and preprocessor use identical projection formulas.
- **Heading Logic**: Validates 0-360° to signed centi-degree conversion and overflow protection.
- **Behavioral Scenarios**: Tests normal movement, GPS jumps, and dead reckoning during outages.
- **Integration**: Verifies stop detection (active stops) against real-world route data.

Run tests using:
```bash
cargo test -p simulator
```
